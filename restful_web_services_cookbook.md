## 1.  使用统一接口
### 如何保持交互的可见性
1.  使用GET获取资源
2.  使用PUT更新资源
3.  使用DELETE删除资源
4.  使用POST执行各种不安全和非幂等的操作
5.  可以添加适当的HTTP标头来描述请求和相应

### 什么是安全性和幂等性
确保客户端可以重复活重试特定的HTTP请求

### 什么时候使用GET
使用GET方法进行安全与幂等的信息获取

### 什么时候使用POST
1.  创建新的资源，将资源作为一个工厂
2.  通过一个控制器资源来修改一个或者多个资源
3.  执行需要大数据输入的查询
4.  在其他HTTP方法看上去不适用的时候，执行不安全或者非幂等性的操作

### 如何使用POST方法创建资源
将一个已存在的资源标识为创建新资源的工厂，虽然您可以将任意资源用作工厂，但是常见的做法是使用一个集合资源


### 什么时候使用PUT方法创建资源
只有在客户端可以决定资源的URI的时候才使用PUT方法创建资源，否则使用POST

## 2.  识别资源
### 2.1 如何从领域名词中识别资源
分析您的用例，找到可以用 创建，读取，更新，删除 动作来操作的领域名词，将每个领域名词都标识位资源，使用POST,GET,PUT,DELETE方法分别为每个资源实现 创建，读取，更新，删除操作

### 2.2 如何选择资源粒度
可以通过网络效率，表述的多少，以及客户端的易用程度帮助确定资源的粒度

### 2.3 如何将资源组织为集合
鲫鱼应用程序特有的条件来识别相似的资源，常见的例子有，共享同一数据库的资源，有相同特性，属性的资源或者客户端看起来相似的资源。

### 2.4 何时将资源合并为复合资源
复合资源，是对其他资源的整合合并
基于客户端的使用模式，性能和延时要求，确定一些新的资源，他们通过聚合其他资源来减少与服务器的交互

## 3. 设计表述
### 3.1 如何使用实体头来注解表述
表述不仅仅是以某种格式序列化后的数据，它是一连串字节加上用于描述那些字节的元数据。
使用一下标头在注解包含消息正文的表述：
1.  Content-Type,用于描述表述类型。eg:charset
2.  Content-Length，用于指定表述正文的字节大小
3.  Content-MD5 提供一致性校验
4.  Content-Encoding 编码表述
5.  Last-Modified 用来说明服务器修改表述或者资源的最后时间

### 3.2 如何解释实体头
1.  Content-Type
2.  Content-Length
3.  Content-Encoding
4.  Content-Language

### 3.3  如何避免字符编码不匹配
使用charset参数
```Content-Type: application/xml;charset=UTF-8```

### 3.4 如何选择表述格式和媒体类型
保持灵活的媒体类型和格式，每个资源都要可以满足多种应用程序用例和客户端需求
```
image/png
application/vnd.ms-excel
application/pdf
```

### 3.5 如何设计XML表述
在每一个表述中，包含一个指向资源本身的self连接，对于那些组成自语言的应用程序领域实体，在表述中要包含他们的标识符号。
如果表述中的某个部分包含自然语言文本，请添加xml:lang属性，表示元素的内容用的是本地化语言

### 3.6 如何设计JSON表述
同xml

### 3.7 如何设计集合表述
每个集合表述包含以下内容：
1.  一个指向集合资源的self连接
2.  如果集合是分页的，并且还有下一页，要有一个指向下一页的连接
3.  如果集合是分页的，并且还有上一页，要有一个指向上一页的连接
4.  一个集合大小的指示符

### 3.8 何时以及如何提供HTML表述
对于那些希望被最终用户使用，应该为他们提供HTML表述，避免为机器客户端设计HTML表述，要为WEB爬虫和此类软件能够正常运作，可以使用微格式或者RDF在标记中注解数据

## 4. 设计URI
### 4.1 如何设计URI
1.  遵循惯性的URI一般容易调试和管理
2.  服务器可以集中代码，以便从请求URI中提取数据
3.  可以避免花费宝贵的设计与实现的时间来发明处理URI的新惯例和规则
4.  通过跨域，子域，和路径来对服务器的URI进行分区，这为您带来负载分配，监控，路由和安全方面的操作提供灵活性

## 5. 查询
### 5.1 如何针对查询设计URI
使用查询参数让客户端来指定过滤条件，排序和投影。将查询参数作为带有合理默认值的可供参数，为了支持常用查询，可以使用预定义的命名查询，编写文档说明每个参数的用途
```
...?sort=aaa&order=desc&limit=10
```
### 5.2 如何设计查询响应
将查询的表述设计为集合资源。如果查询没有匹配到资源，返回一个空集合

### 5.3 如何支持有大量输入的查询请求
使用POST来支持大查询

### 5.4 如何存储查询
当客户端使用POST发起一个查询请求时，创建一个新资源，他的状态中包含查询条件，返回一个带Location头的响应码201，Location指向创建的资源，实现一个针对新资源的GET请求。返回查询结果。
如果客户端或另一个客户端用POST发起相同的查询请求，找到匹配请求的资源，客户端被重定向到该URI上。
